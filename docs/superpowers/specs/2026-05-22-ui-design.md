# UI Design Spec — pbai
**Date:** 2026-05-22  
**Status:** Approved

## Overview

pbai è un'app mobile-first per il tracciamento vocale delle spese. Il design segue la direzione **"warm minimalism"**: sfondo caldo, tipografia serif italiana per i numeri, sans-serif neutro per il testo UI. Nessun framework esterno (no MUI) — design system implementato come estensione Tailwind.

| `accent` | `#b07d48` | €, bottone OK, chip selezionato |
| `expense` | `#c0392b` | Segno −, switch spesa selezionato |
| `income` | `#27ae60` | Segno +, switch entrata selezionato (futuro) |

### Tipografia

| Ruolo | Font | Stile | Uso |
|-------|------|-------|-----|
| Display | DM Serif Display | italic 400 | Importo, domanda di clarification |
| UI | Inter | 300–600 | Tutto il resto |

Font caricati via Google Fonts in `index.html`.

### Formato importo

```
[segno]  [intero]  [decimali]  [€]
```

- **Segno** (`−` o `+`): Inter, `font-size: 0.58em`, colore `expense` o `income`, `vertical-align: middle`
- **Intero**: DM Serif Display italic, size base (~64px su mobile)
- **Decimali** (`.XX`): DM Serif Display italic, `font-size: 0.48em`, `vertical-align: super`
- **€**: Inter normal, `font-size: 0.21em`, colore `accent`, `vertical-align: super`

Esempio renderizzato: `−12.00€` (con decimali più piccoli e € in ambrato)

## Componenti

### ModeSwitch (nuovo)

Switch − / + in cima alla schermata, sempre visibile.

**Struttura:**
```
[track: ede3d6 pill]
  [− pill]  [+ pill]
```

**Stati per ogni pill:**
- **Selezionato −**: `background: rgba(192,57,43,.12)`, `border: 1.5px solid #c0392b`, `color: #c0392b`
- **Selezionato +**: `background: rgba(39,174,96,.12)`, `border: 1.5px solid #27ae60`, `color: #27ae60`
- **Non selezionato**: sfondo trasparente, bordo trasparente, `color: #c9b8a8`

**Dimensioni:** pill 32×32px, bordo-raggio 50%, font 16px Inter 500.

**Default:** − (spesa) selezionato.

**Comportamento:** il valore del mode switch viene passato al parser API come campo `mode: 'expense' | 'income'`. Questo consente all'utente di dire "caffè, 1 euro" senza dover specificare "ho speso".

**Persistenza:** il modo si resetta a `expense` ad ogni reset di sessione.

---

### TransactionPreview

Mostra titolo e importo dell'operazione parsata.

```
[titolo — Inter 10px, uppercase, tracked, color: muted]
[separatore — 28px, color: border]
[importo — formato descritto sopra]
```

Animazione `fade-up` all'apparizione (opacity 0→1, translateY 10px→0, 350ms ease-out).

---

### TagChips

Chip toggle per i tag suggeriti dall'AI.

- **Selezionato:** `background: rgba(176,125,72,.10)`, `border: 1px solid rgba(176,125,72,.35)`, `color: accent`
- **Deselezionato:** bordo `border`, testo `dim`, `text-decoration: line-through`
- Font: Inter 9px, uppercase, tracked, `border-radius: 20px`

---

### ClarificationPrompt

Domanda di chiarimento dell'AI.

- Domanda: DM Serif Display italic, ~36px, colore `text`
- Hint ("Tieni premuto…"): Inter 9px, uppercase, tracked, colore `dim`

Animazione `fade-up` all'apparizione.

---

### MicButton

Bottone microfono principale.

| Stato | Stile |
|-------|-------|
| `idle` | `bg: surface`, bordo `border`, icona `dim` |
| `recording` | `bg: expense`, bordo `accent/50`, icona bianca, `animate-red-breathe` |
| `processing` | `bg: surface`, bordo `border`, opacità 50%, arco SVG rotante in `accent` |
| `preview/clarification` | come idle (non mostrato — sostituito dai bottoni) |

**Dimensioni:** 96×96px (w-24 h-24), wrapper 112×112px per l'arco di processing.

---

### Bottoni Cancel / OK

Mostrati in coppia al posto del mic in stato `preview` e `clarification`.

- **Cancel:** cerchio 52×52px, bordo `border` 1.5px, icona `dim`. Solo visibile, mai disabled.
- **OK:** cerchio 52×52px, bordo `accent` 1.5px, icona `accent`. Disabled (bordo `border`, icona `dim`, opacity 30%) in stato `clarification`.

---

### App Layout

```
[status bar — dot + label, 16px height]
[mode switch — centrato]
[content area — flex-1, centrato verticalmente]
  → idle:          hint text
  → recording:     dot animato + "In ascolto"
```

## Implementazione

**Stack:** invariato — React + Tailwind + nessun nuovo pacchetto.

**Keyframes da aggiungere a `tailwind.config.ts`:**
```js
'fade-up':       { from: { opacity:'0', transform:'translateY(10px)' }, to: { opacity:'1', transform:'translateY(0)' } }
'red-breathe':   { '0%,100%': { boxShadow:'0 0 12px rgba(192,57,43,.8), 0 0 28px rgba(192,57,43,.3)' }, '50%': { boxShadow:'0 0 24px rgba(220,70,50,1), 0 0 52px rgba(220,70,50,.5)' } }
'arc-spin':      { to: { transform:'rotate(360deg)' } }
```

**Approccio:** estendere `tailwind.config.ts` con i token sopra (`colors.pbai.*`, `fontFamily.display`, `fontFamily.ui`, `keyframes`, `animation`). Google Fonts via `<link>` in `index.html`.

**Nuovo componente:** `ModeSwitch.tsx` — stato locale + callback `onChange(mode)` a `App.tsx`. Il mode viene passato come parametro aggiuntivo a `parse()`.

**Aggiornamento API:** il campo `mode: 'expense' | 'income'` viene aggiunto alla request del parser (`parse({ text, partial, mode })`). Il parser lo usa come contesto per capire se l'operazione è una spesa o un'entrata senza che l'utente debba dirlo.

**Convenzione `importo`:** il valore in IndexedDB rimane sempre positivo. Il segno (−/+) è esclusivamente display, determinato dal `mode` al momento della registrazione. Quando il mode `income` sarà attivo, il campo `mode` andrà salvato anche nella transazione (fuori scope per ora).

**Test:** aggiornare `TransactionPreview.test.tsx` — il nuovo formato dell'importo è `−1.50€` (non più `€ 1.50`). Aggiungere `ModeSwitch.test.tsx`.
